# C05 — ADR Required Constraint

## 목적

중요한 아키텍처 결정사항을 기록으로 남겨 미래의 개발자가 "왜"를 이해하도록 합니다.

## ADR (Architecture Decision Record)

ADR은 다음을 포함하는 문서입니다:
- **문제**: 무엇을 결정해야 했는가?
- **고려사항**: 어떤 선택지가 있었는가?
- **결정**: 어떤 선택을 했는가? 왜?
- **결과**: 긍정적/부정적 영향은?

## 활성 Phase

production (프로덕션 환경에만 필수)

## 템플릿

```markdown
---
status: ACCEPTED|REJECTED|DEPRECATED
decision_date: 2026-04-29
---

# ADR-001: 데이터 저장소로 PostgreSQL 선택

## 문제
데이터 저장소 기술 선택

## 고려사항
1. PostgreSQL: 안정적, JSONB 지원, 오픈소스
2. MongoDB: 유연한 스키마, NoSQL
3. DynamoDB: AWS 관리형, 비용

## 결정
PostgreSQL을 선택

**이유**:
- 관계형 데이터에 적합
- 풍부한 생태계
- 무료 오픈소스

## 결과
- 긍정: 확장성, 안정성
- 부정: 스케일-아웃 복잡도
```
