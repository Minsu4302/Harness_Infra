# C04 — Commit Gate Constraint

## 목적

모든 커밋이 일관된 메시지 형식을 따르도록 보장합니다.

## 커밋 메시지 형식

```
type(scope): description
```

### type 유효 값

| 타입 | 설명 | 예시 |
|------|------|------|
| feat | 새 기능 | feat(auth): 로그인 기능 추가 |
| fix | 버그 수정 | fix(api): 응답 시간초과 문제 해결 |
| refactor | 코드 개선 | refactor(utils): 함수 분리 |
| docs | 문서 추가 | docs(readme): 설치 가이드 추가 |
| test | 테스트 작성 | test(auth): 로그인 테스트 추가 |
| chore | 설정 변경 | chore(deps): 의존성 업데이트 |

### scope (선택사항)

기능 영역을 명시합니다:
- auth, api, ui, db, config, infra 등

## 활성 Phase

stabilization, production

## 체크 항목

- ✓ 첫 줄이 `type(scope): description` 형식인가?
- ✓ type이 유효한 값인가?
- ✓ 설명이 명확하고 구체적인가?
- ✓ 첫 글자가 소문자인가?

## 모범 사례

### 좋은 예

```
feat(auth): OAuth 2.0 로그인 구현

로그인 페이지에 Google/GitHub 소셜 로그인 추가
- OAuth 2.0 흐름 구현
- 토큰 저장소 추가
- 통합 테스트 작성

Closes #123
```

### 나쁜 예

```
❌ 기능 추가        (타입 없음)
❌ FEAT(AUTH): ...  (대문자)
❌ fix bug          (너무 모호함)
❌ 버그를 수정했습니다  (한국어)
```
