# C01 — Harness Size Constraint

## 목적

각 스크립트 파일이 1000줄을 초과하지 않도록 유지합니다.

복잡한 로직은 함수로 분리하고, 너무 큰 파일은 모듈로 나누어야 합니다.

## 기준

| 항목 | 기준 |
|------|------|
| 최대 줄 수 | 1000줄 |
| 적용 대상 | scripts/*.sh, linters/*.sh, validators/*.sh |
| 활성 Phase | development, stabilization, production |

## 체크 항목

- ✓ 각 스크립트 파일이 1000줄 이하인가?
- ✓ 함수 단위로 코드가 모듈화되어 있는가?
- ✓ 주석과 공백줄을 포함한 전체 줄 수인가?

## 위반 시 개선 방법

### 1. 함수 분리
큰 스크립트는 작은 함수로 나누기:

```bash
# 나쁜 예: 모든 로직이 main에 있음 (700줄)
# ❌ constraint-check.sh

# 좋은 예: 함수로 분리
# ✅ constraint-check.sh + linters/_lib.sh
```

### 2. 모듈 분할
200~300줄 이상의 로직은 별도 파일로 분리:

```bash
# 나쁜 예
scripts/gc-agent.sh (600줄)

# 좋은 예
scripts/gc-agent.sh (200줄)
scripts/_gc-scan.sh (200줄)
scripts/_gc-collect.sh (200줄)
```

## 측정

```bash
# 현재 상태 확인
bash linters/harness-size.sh

# 파일별 줄 수 확인
wc -l scripts/*.sh linters/*.sh
```
