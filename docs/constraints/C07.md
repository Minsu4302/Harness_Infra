# C07 — Session Checkpoint Constraint

## 목적

세션이 너무 길어지거나 컨텍스트가 과다해지는 것을 방지합니다.

세 가지 조건 중 하나라도 충족되면 **체크포인트**를 강제합니다.

## 트리거 조건 (OR 관계)

### 조건 1: 경과 시간 초과
- 기한: HARNESS.md의 `max_hours` (기본값: 4시간)
- 의미: 한 세션이 너무 길어졌음

### 조건 2: 예산 경고 연속 초과
- 기한: HARNESS.md의 `budget_warn_streak` (기본값: 3회)
- 의미: 컨텍스트 메모리 누적

### 조건 3: 미결 항목 과다
- 기한: HARNESS.md의 `max_open_questions` (기본값: 5개)
- 의미: 미해결 사항이 쌓이고 있음

## 활성 Phase

development, stabilization, production (planning 제외)

## 체크포인트 수행 방법

체크포인트 조건이 트리거되면:

1. **현재 상태 저장**
   ```bash
   # open-questions.md를 검토하고 정리
   # task.md를 완료 상태로 갱신
   # logs에 최종 메트릭 기록
   ```

2. **history에 백업**
   ```bash
   # task.md → .harness/reports/history/task_YYYYMMDD_HHMMSS.md
   # 모든 open questions 정리
   ```

3. **새 세션 시작**
   ```bash
   bash scripts/context-loader.sh
   ```

## 측정

```bash
# C07 실행
bash linters/session-checkpoint.sh

# 조건별 현재 상태 확인
grep "^started_at:" .harness/session/task.md        # 조건 1
grep "budget_warn_streak" .harness/reports/last-check.json  # 조건 2
grep -c "^- \[" .harness/session/open-questions.md  # 조건 3
```

## 모범 사례

### 체크포인트가 필요 없는 경우
```
✓ 세션 시작 후 < 4시간
✓ context budget < 70%
✓ open questions < 3개
```

### 체크포인트가 필요한 경우
```
✗ 세션 시작 후 4시간 초과
✗ budget_warn_streak 3회 연속
✗ open questions 5개 초과
```

## 체크포인트 예시

```bash
# 1. 현재 상태 확인
bash linters/session-checkpoint.sh
# 출력: FAIL:C07:체크포인트 필요 (조건 1개 충족)

# 2. task.md 정리
# - 미완료 항목을 open-questions.md로 이동
# - completed 상태로 표시

# 3. history에 백업
ls -la .harness/reports/history/

# 4. context 갱신
bash scripts/context-loader.sh

# 5. 새 작업 시작
# task.md를 새로 작성하고 시작
```
