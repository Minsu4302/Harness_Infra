# C06 — GC Frequency Constraint

## 목적

기술 부채와 컨텍스트가 정기적으로 정리되도록 보장합니다.

## GC (Garbage Collection)

GC는 다음을 수행합니다:
1. 기술 부채 스캔 → debt-report.md 생성
2. 로그 수집 → logs/에 메트릭 저장
3. 컨텍스트 정리

## 활성 Phase

planning (계획 단계에서 한 번은 실행)

## 권장 주기

| Phase | 권장 주기 |
|-------|---------|
| planning | 프로젝트 시작 시 1회 |
| development | 1-2주에 1회 |
| stabilization | 주 1회 |
| production | 일 1회 |

## 측정

```bash
# GC 실행 (스캔 + 로그 수집)
bash scripts/gc-agent.sh --scan --collect

# 결과 확인
ls -la logs/events/
cat .harness/reports/debt-report.md
```

## 결과 검증

- ✓ logs/에 최신 JSONL 파일이 생성되었는가?
- ✓ debt-report.md가 업데이트되었는가?
- ✓ gc_scan_complete 이벤트가 기록되었는가?
