# Product Principles

## 목적

하네스 엔지니어링 인프라가 추구하는 핵심 가치와 설계 원칙을 정의합니다.

## 핵심 원칙

### 1. 컨텍스트 제어 우선

**원칙**: 에이전트가 어떤 정보를 가지고 있는지가 출력 품질을 결정한다.

**구현**:
- context-loader.sh가 세션 시작 시 모든 필요 정보를 로드
- 컨텍스트 예산(2000줄)을 엄격히 관리
- 정기적 GC로 부실한 정보 제거

### 2. 명시적 제약 강제

**원칙**: 규칙은 지침이 아니라 자동으로 검증되는 제약이어야 한다.

**구현**:
- 모든 제약(C01~C07)은 스크립트가 검증
- 커밋 전 자동으로 실행되어 위반 방지
- 실패 시 구체적인 원인 제시

### 3. 피드백 루프 가시화

**원칙**: 문제는 빠르게 감지되고, 해결 경로가 명확해야 한다.

**구현**:
- logs/에 구조화된 메트릭/이벤트 기록
- debt-report.md로 누적된 문제 시각화
- 체크포인트(C07)로 세션 과부하 조기 감지

## 설계 철학

| 철학 | 의미 |
|------|------|
| **명시성** | 암묵적 규칙 대신 명시적 선언 |
| **자동화** | 수동 검증 대신 자동화 스크립트 |
| **추적성** | 모든 결정과 이력을 문서화 |
| **회귀 방지** | 과거 실수를 감지하는 테스트 |

## 성공 지표

| 지표 | 목표 |
|------|------|
| 컨텍스트 예산 사용률 | < 70% |
| 제약 위반율 | 0% |
| 평균 세션 길이 | < 4시간 |
| 기술 부채 해결율 | > 80%/분기 |
